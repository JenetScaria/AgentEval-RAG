# AgentEval RAG

A self-evaluating agentic RAG (Retrieval-Augmented Generation) system built on Amazon product reviews. The pipeline retrieves relevant context, generates grounded answers with citations, scores its own output using RAGAS metrics, and automatically retries with web fallback when quality drops below threshold.

---

## How it works

```
Query
  │
  ▼
Router ──────────────────────────────────────────────────────────┐
  │ classifies query as simple / multi_hop / web                 │
  ▼                                                              │
Retrieval                                                        │
  │ FAISS (dense, 60%) + BM25 (sparse, 40%) + cross-encoder     │
  ▼                                                              │
Generation                                                       │
  │ Gemini answers strictly from retrieved context + citations   │
  ▼                                                              │
Evaluation                                                       │
  │ RAGAS scores: faithfulness · context precision · relevancy   │
  │                                                              │
  ├── score ≥ threshold ──────────────────────────────► Answer  │
  │                                                              │
  └── score < threshold ──► Re-retrieval ──────────────────────►┘
                             (web on retry 1, local on retry 2)
```

---

## Features

- **Hybrid retrieval** — FAISS semantic search fused with BM25 keyword search, reranked by a cross-encoder
- **Automatic quality evaluation** — RAGAS metrics (faithfulness, context precision, answer relevancy) scored after every response
- **Self-healing retry loop** — switches to live web search when local retrieval score is too low
- **Inline citations** — every answer references the exact chunks it used
- **Experiment tracking** — all queries, scores, and retry metadata logged to MLflow
- **Two interfaces** — Streamlit UI for interactive use, FastAPI for programmatic access
- **Free to run** — uses Gemini 2.0 Flash (free tier, no credit card required)

## Tech Stack

| Layer | Technology |
|---|---|
| LLM | Gemini 2.0 Flash (`google-genai`) |
| Orchestration | LangGraph `StateGraph` |
| Dense retrieval | FAISS `IndexFlatIP` + `sentence-transformers` |
| Sparse retrieval | BM25Okapi (`rank-bm25`) |
| Reranking | Cross-encoder (`sentence-transformers`) |
| Evaluation | RAGAS + heuristic fallback |
| Experiment tracking | MLflow |
| Web search | Tavily → DuckDuckGo fallback |
| UI | Streamlit |
| API | FastAPI + Uvicorn |
| Config | Pydantic Settings + `.env` |
| Dataset | McAuley-Lab/Amazon-C4 (HuggingFace) |

## Quickstart

### 1. Clone and install

```bash
git clone <your-repo-url>
cd agenteval_rag
pip install -r requirements.txt
```

### 2. Get a free Gemini API key

1. Go to [aistudio.google.com](https://aistudio.google.com)
2. Sign in with your Google account
3. Click **Get API key** — no credit card required
4. Copy the key

### 3. Configure environment

```bash
cp .env.example .env
```

Open `.env` and set your key:

```env
GEMINI_API_KEY=AIzaSy...your_key_here
```

### 4. Build the indexes

Downloads 5,000 Amazon reviews and builds the FAISS + BM25 indexes. Only needs to run once.

```bash
python src/utils/ingest.py --samples 5000
```

Use `--samples 10000` for a larger, richer knowledge base.

### 5. Launch

**Streamlit UI:**
```bash
streamlit run app.py
```

**FastAPI:**
```bash
uvicorn api:app --reload
```
API docs available at `http://localhost:8000/docs`

## Configuration

All settings can be overridden via `.env`. Defaults shown below.

```env
# Required
GEMINI_API_KEY=your_key_here

# Model
GEMINI_MODEL=gemini-2.0-flash

# Retrieval
TOP_K_RETRIEVAL=10      # chunks fetched before reranking
TOP_K_RERANK=5          # chunks that survive reranking
CHUNK_SIZE=512          # characters per chunk
CHUNK_OVERLAP=50        # overlap between consecutive chunks

# Quality thresholds (0.0 – 1.0)
# answers scoring below these on any metric trigger a retry
FAITHFULNESS_THRESHOLD=0.7
CONTEXT_PRECISION_THRESHOLD=0.7
ANSWER_RELEVANCY_THRESHOLD=0.7
MAX_RETRIES=2

# Optional — web search (DuckDuckGo used as free fallback)
TAVILY_API_KEY=your_tavily_key_here

# MLflow
MLFLOW_TRACKING_URI=mlruns
MLFLOW_EXPERIMENT_NAME=agenteval_rag
```

## Evaluation Metrics

After every answer the pipeline scores itself on three dimensions:

| Metric | What it measures | Triggers retry if… |
|---|---|---|
| **Faithfulness** | Does the answer only use facts from the retrieved context? | Answer contains hallucinated content |
| **Context Precision** | Were the retrieved chunks actually relevant to the question? | Wrong chunks were retrieved |
| **Answer Relevancy** | Does the answer directly address the question? | Answer drifts off-topic |

Scores are computed by RAGAS. If RAGAS is unavailable, the system falls back to token-overlap heuristics so the pipeline never crashes.

## API Reference

### `POST /query`

```json
{
  "query": "What do customers say about HEPA filters for allergies?"
}
```

Response:

```json
{
  "answer": "Customers report significant improvement in allergy symptoms [1]...",
  "citations": ["amazon_c4"],
  "query_type": "simple",
  "eval_scores": {
    "faithfulness": 0.87,
    "context_precision": 0.81,
    "answer_relevancy": 0.79,
    "overall": 0.82
  },
  "retry_count": 0,
  "used_web": false
}
```

### `POST /ingest`

Triggers the ingestion pipeline programmatically.

```json
{ "samples": 5000 }
```

### `GET /health`

Returns `{ "status": "ok" }` — use for uptime checks.
